# Intake Interview Guide

Use this framework during Phase 1 to gather comprehensive context about the application under review.

## Core Interview Questions

### 1. Application Overview
- "What does this application do? Give me the elevator pitch."
- "What type of application is this?" (web app, CLI tool, mobile app, desktop app, API)
- "How do users access it?" (URL, install, package manager, app store)

### 2. Target Users & Segments
- "Who are your primary users? Describe 2-3 distinct user types."
- "What's their typical technical background?"
- "Are there internal vs. external users with different needs?"
- "What's the rough user base size?" (10s, 100s, 1000s, millions)

### 3. Key Workflows
- "What are the 3-5 most important tasks users perform?"
- "Walk me through the critical path — the single most important user journey."
- "Are there any onboarding or first-time user flows?"
- "What workflows do users complain about most?"

### 4. Known Pain Points
- "What UX issues are you already aware of?"
- "What do users complain about most frequently?"
- "Are there any areas where users frequently get stuck or confused?"
- "What's your current support ticket volume, and what are the top categories?"

### 5. Business Goals
- "What business outcomes should this UX review support?"
- "Are there conversion, retention, or engagement metrics you're trying to improve?"
- "Are there upcoming launches, redesigns, or migrations that this review should inform?"

### 6. Competitive Landscape
- "Who are your main competitors?"
- "What do competitors do better or worse in terms of UX?"
- "Are there any competitor features your users frequently request?"

### 7. Success Metrics
- "How will you measure whether UX improvements were successful?"
- "What KPIs matter most?" (task completion rate, time-on-task, error rate, NPS, retention)
- "Is there a timeline or deadline for implementing changes?"

### 8. Technical Constraints
- "Are there technical constraints that limit UX changes?" (legacy systems, API limitations)
- "What's your tech stack?" (relevant for feasibility of recommendations)
- "Are there accessibility requirements?" (WCAG level, regulatory compliance)

### 9. Review Scope
- "Are there specific areas or features you want us to focus on?"
- "Are there areas you'd like us to skip or deprioritize?"
- "Do you have existing personas, user research, or analytics we should reference?"

## Interview Conduct Guidelines

- Ask questions conversationally, not as a checklist
- Follow up on interesting answers — dig into specifics
- Note emotional language — it signals strong opinions and pain points
- Summarize understanding back to the user for confirmation
- Skip questions where the answer is already clear from context
- Adapt question depth to the user's available time
- Record answers in `.ux-review/interviews/intake.md`

## Minimum Viable Intake

If the user is short on time, these 4 questions provide enough context to proceed:

1. What does the app do and who uses it?
2. What are the 3 most important user tasks?
3. What UX problems are you already aware of?
4. What areas should we focus on?
